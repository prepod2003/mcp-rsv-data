# MCP:RSV Data

**Бесплатный MCP-сервер для работы с данными рабочей базы 1С:Предприятие.** Даёт ИИ-агенту
(Claude, Cursor и др.) читать данные базы простыми словами — остатки, документы, справочники,
регичстры. Агенту **не нужно знать язык запросов 1С**: он описывает, что хочет, а сервер строит
запрос сам. Работает на **любой конфигурации** (УТ, ERP, БП, самописная) и отдаёт только то,
что доступно текущему пользователю на чтение. Реализован как расширение конфигурации 1С — только
чтение, базу не меняет.

> Это про **данные**. Если нужна **разработка** в 1С:EDT (создание и редактирование кода, метаданных,
> форм, СКД, отладка) — это другой, **платный** продукт линейки RSV:
> **[MCP:RSV Server](https://prepod2003.github.io/edt-ai-1c/)**.

## Быстрый старт

1. **Установить расширение в базу.** В 1С:Предприятии: функции технического специалиста →
   «Управление расширениями конфигурации» → «Добавить» → выберите `RSVData.cfe` → перезапустите
   программу. (Или из Конфигуратора: Конфигурация → Расширения → загрузить из файла → обновить
   конфигурацию базы F7.)
2. **Подключить ИИ-клиент** — двумя способами:
   - **HTTP-сервис** (если база опубликована на веб-сервере) — клиент обращается к 1С напрямую;
     `rsvdata-bridge.exe` помогает собрать готовый блок настроек, но не обязателен (на Linux/macOS
     блок пишется вручную).
   - **Локальный мост** (если база не опубликована) — `rsvdata-bridge.exe` подключается к базе по COM
     (Windows + установленная платформа 1С).
3. Запустите `rsvdata-bridge.exe`, выберите способ, укажите параметры базы и клиент — мастер
   проверит связь, выдаст готовый блок настроек и при желании сам впишет его в конфиг клиента.

Подробно (оба способа, примеры конфигов для Claude Code / Cursor / Cline, диагностика) —
в [`guide.md`](guide.md).

## Инструменты

`config` (паспорт базы) · `describe` (что есть в базе) · `get_structure` (полная структура объекта) ·
`query` (чтение данных без языка запросов) · `execute_query` (готовый запрос 1С) · `ping` · `help`.

Описание каждого — в [`guide.md`](guide.md), а также внутри самого сервера: `help` без параметров —
обзор, `help topic=<инструмент>` — подробности.

## 🔧 Разработка в 1С:EDT — MCP:RSV Server (платный)

MCP:RSV Data **читает** данные. Если нужно **разрабатывать** в 1С:EDT — создавать и править BSL-код,
объекты метаданных, формы, отчёты СКД, отлаживать — для этого есть профессиональный продукт линейки RSV:

### 👉 MCP:RSV Server — [prepod2003.github.io/edt-ai-1c](https://prepod2003.github.io/edt-ai-1c/)

Кстати: само расширение MCP:RSV Data **от начала до конца создано ИИ-агентом именно через MCP:RSV
Server** — создание расширения и его объектов (HTTP-сервис, общие модули), написание всего кода и его
проверка через код-ревью. Человек-программист не написал вручную ни строки. То есть сложную разработку
в 1С сегодня реально вести полностью силами ИИ (подробнее — раздел P.S. в [`guide.md`](guide.md)).
